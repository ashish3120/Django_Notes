## 1. Introduction
The Django Template Engine is a string-based language that allows you to separate the **Presentation Layer** (HTML) from the **Business Logic** (Python). It allows the backend to send data to the frontend via a **Context Dictionary**.

> **Pro-Tip:** Keep your templates "dumb." Heavy logic should remain in `views.py`; templates should only handle how data is displayed.

---

## 2. Template Syntax
Django uses specific delimiters to process logic vs. plain text:

| Type | Syntax | Description |
| :--- | :--- | :--- |
| **Variables** | `{{ variable }}` | Outputs the value from the context. |
| **Tags** | `{% tag %}` | Handles logic (loops, blocks, etc.). |
| **Filters** | `{{ val\|filter }}`| Modifies the variable for display. |
| **Comments** | `{# comment #}` | Internal notes, not rendered in HTML. |

---

## 3. Dynamic Rendering
To pass data from the backend, we use the `render()` function in `views.py`:

```python
def student_list(request):
    data = [
        {'name': 'Abhijit', 'age': 26},
        {'name': 'Sandeep', 'age': 22}
    ]
    return render(request, 'students.html', {'students': data})
